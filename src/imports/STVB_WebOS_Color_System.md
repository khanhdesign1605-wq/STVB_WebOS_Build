# 🎨 Color System — Style Guide

> Hệ thống màu sắc chuẩn hóa cho WebOS. Tất cả màu được tổ chức theo thang độ (scale) từ **025 → 900**, phân nhóm theo ngữ nghĩa (semantic) để đảm bảo tính nhất quán xuyên suốt toàn bộ giao diện.

## 2. Semantic Color Tokens (Token ngữ nghĩa)

Các token ngữ nghĩa **không gắn với giá trị hex trực tiếp**, mà tham chiếu đến palette gốc. Đây là lớp abstraction giúp dễ dàng thay đổi theme mà không cần sửa từng component.

### 🎯 Call to Action Colors

Dùng cho nút bấm, liên kết hành động và các yếu tố tương tác chính.

| Token | Tham chiếu | Mô tả |
|-------|-----------|-------|
| `action` | `Primary/900` | Nền nút CTA mặc định |
| `action-hover` | `Primary/800` | Nền nút CTA khi hover |
| `action-press` | `Primary/900` | Nền nút CTA khi nhấn (với hiệu ứng shadow) |
| `primary-light` | `Primary/100` | Nền nút secondary / ghost |
| `primary-light-hover` | `Primary/300` | Nền nút secondary khi hover |
| `primary-light-press` | `Primary/400` | Nền nút secondary khi nhấn |
| `emphasis-gray` | `Neutral/800` | Nhấn mạnh bằng màu trung tính (icon, label) |

---

### 🗂️ Common Colors

Dùng cho các thành phần giao diện chung: layout, text, divider, container.

| Token | Tham chiếu | Mô tả |
|-------|-----------|-------|
| `common-item` | `Neutral/800` | Màu chữ và icon chính |
| `common-text` | `Neutral/600` | Màu chữ phụ (subtitle, label, placeholder) |
| `common-line` | `Neutral/300` | Màu viền, đường kẻ phân cách (divider) |
| `common-contain` | `Neutral/050` | Nền container, card, sidebar |
| `common-hover` | `Neutral/200` | Nền khi hover vào item trong list |
| `common-press` | `Neutral/400` | Nền khi nhấn vào item trong list |

## 4. Hướng dẫn sử dụng (Usage Guidelines)

### ✅ Nên làm

- **Luôn dùng semantic token** (`--color-action`, `--color-common-text`) trong component code — không hardcode hex.
- **Dùng shade 500–700** cho icon và text có nội dung quan trọng để đảm bảo tương phản đủ WCAG AA.
- **Dùng shade 025–100** cho nền container, badge, chip trạng thái để tránh quá nặng về thị giác.
- **Kết hợp `*-text` + `*-container`** của cùng nhóm màu để tạo badge/alert nhất quán.

### ❌ Không nên làm

- Không dùng màu palette trực tiếp (vd: `#F04438`) trong component — luôn đi qua token.
- Không dùng màu Error/Warning/Success cho mục đích trang trí — chỉ dùng đúng ngữ nghĩa.
- Không dùng shade quá nhạt (025–100) cho chữ trên nền trắng — tương phản sẽ không đạt chuẩn.
- Không kết hợp màu khác nhóm ngữ nghĩa (vd: chữ màu Error trên nền Success).
